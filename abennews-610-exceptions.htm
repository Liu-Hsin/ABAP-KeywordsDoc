Exception Handling in Release 6.10

Since Release 6.10 it has been possible to catch class-based exceptions in TRY...CATCH... ENDTRY blocks. This replaces exception handling using either catchable runtime errors or the EXCEPTIONS parameters of function modules and methods.

The new keyword CLEANUP has been introduced for a program area that helps to create a consistent state after an exception or runtime error occurs within the protected area. Across the call hierarchy, runtime errors are caught at a higher level, so that if a method terminates, for instance, a consistent object state remains and global resources are released in accordance with regulations.

For a detailed description of this topic, see Exception Handling.
