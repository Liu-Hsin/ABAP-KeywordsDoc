Conditions for the Strict Mode

The strict mode of the syntax check in Release 7.55 applies to all ABAP SQL statements that use one of the following attributes introduced in Release 7.55:

  • Using function ALLOW_PRECISION_LOSS
  • Optional addition window frame specification
  • Window functions FIRST_VALUE and LAST_VALUE.
  • Use of one of the new date/time functions TSTMPL_TO_UTCL, TSTMPL_FROM_UTCL, DATS_TO_DATN, DATS_FROM_DATN, TIMS_TO_TIMN, TIMS_FROM_TIMN, UTCL_CURRENT, UTCL_ADD_SECONDS, UTCL_SECONDS_BETWEEN, DATN_ADD_DAYS, DATN_ADD_MONTHS, and DATN_DAYS_BETWEEN.
  • New additions after the ORDER BY clause: NULLS FIRST and NULLS LAST.
  • Use of one of the new aggregate functions MEDIAN, STDDEV, VAR, CORR, and CORR_SPEARMAN.
  • New geometry conversion function as_geo_json.
  • Subqueries in a relational expression using the operator IN with an operand list.
  • Use of the window function NTILE
  • Use of the SQL functions TO_CLOB or TO_BLOB
  • Use of typed literals
  • Use of the string functions REPLACE_REGEXPR, LIKE_REGEXPR, and OCCURRENCES_REGEXPR.
  • Use of set indicators after UPDATE FROM.

Rules for the Strict Mode

The strict mode in Release 7.55 covers all rules of the strict mode in Release 7.54.
