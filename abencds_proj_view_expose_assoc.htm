CDS DDL - projection_view, expose_assoc

Syntax

... {association[ [filter]] [AS alias] [: REDIRECTED TO projection_view]}
  | {composition[ [filter]] [AS alias] [: REDIRECTED TO COMPOSITION CHILD projection_view]}
  | {to-parent association[ [filter]] [AS alias] [: REDIRECTED TO PARENT projection_view] }

Effect

You can expose an association from the projected entity in the element list of your CDS projection view. You can use the association in a path expressions to expose individual fields from the association target. And there are several additions you can use with an exposed association:

  • An exposed association in a CDS projection view can be modified. That means that you can add a filter and in this way implicitly change the cardinality. Then you can assign a new alias name to the modified association. Find details in the topic CDS DDL - projection_view, modified association below.
  • If the original association target is also projected, you can redirect the association to the new target using the keyword REDIRECTED TO. Find details in the topic CDS DDL - projection_view, REDIRECTED TO below.

All elements included in the ON clause of an exposed CDS association must also be included in the element list of the CDS projection view.

Hint

  • Projected CDS associations, CDS compositions, and CDS to-parent associations can be specified as values for the CDS annotations ElementRef and AssociationRef.
