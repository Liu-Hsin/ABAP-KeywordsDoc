release contract

Classification of a repository object that controls its usability as an API. There are currently three different release contracts: C0, which allows enhancement fields to be added at specified enhancement points, C1, which allows a stable interface for use within the AS ABAP, and C2, which also guarantees a stable interface for use outside the AS ABAP. Objects are classified in ADT or using special transactions such as SCFD_REGISTRY.
