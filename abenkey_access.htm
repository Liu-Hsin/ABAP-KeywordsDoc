Effect

Specifies a table key as a search key. The primary table key or a secondary table key can be specified. The values can be declared either implicitly in a work area wa behind FROM or explicitly by listing the components of the table key behind TABLE KEY.

When the primary table key is used, the table categories are accessed as follows:

  • Standard tables are searched linearly.
  • Sorted tables are searched binarily.
  • Hashed tables are searched using the hash algorithm.

When the secondary table key is used, a binary search is used for a sorted key and a hash algorithm for a hash key.
