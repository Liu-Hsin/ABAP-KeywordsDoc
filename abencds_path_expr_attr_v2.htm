CDS DDL - CDS View Entity, path_expr, attributes

Syntax

... [ [1|*:] [INNER|{LEFT OUTER}] [WHERE] [cds_cond] ] ...

Effect

For every CDS association _assoc of a path expression of a CDS view entity, attributes for this part of the path expression can be defined in square brackets [ ]. The following attributes can be specified:

  • With 1|*:, the cardinality of the section can be specified.
  • With INNER|{LEFT OUTER} the join type can be specified.
  • With [WHERE] cds_cond a filter condition cds_cond can be specified.

Example

The CDS view entity contains path expressions with filter conditions in the FROM clause, in the SELECT list, and in the WHERE clause.

@AccessControl.authorizationCheck: #CHECK
define view entity DEMO_SALES_CDS_PATH_EXPR_1
  as select from
    DEMO_SALES_CDS_SO_I._SalesOrder  
      [lifecycle_status <> 'C' and lifecycle_status <> 'X']
    association [0..1] to demo_so_inv_he as _invoice_header
      on DEMO_SALES_CDS_SO.id = _invoice_header.buyer_id
    {
      key so_key as soKey,
          lifecycle_status,
          _invoice_header.dunning_level,
          demo_sales_cds_so._Buyer,
          DEMO_SALES_CDS_SO[inner].buyer_id,
          DEMO_SALES_CDS_SO.id
    }
    where
      _invoice_header.dunning_level > '0' and
      _Items[1:currency <>'EUR'].material = 'WOOD'
